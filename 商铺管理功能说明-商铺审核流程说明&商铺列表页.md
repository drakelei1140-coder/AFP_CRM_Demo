## 商铺管理功能说明

### 商铺审核流程说明

#### 审核流程适用说明

商铺（SID）的审核流程属于系统底层通用流程，需兼容不同通道场景。\
是否进入风控审核环节，取决于当前商铺所选择的服务通道：

*   当所选通道中包含 **非 AFP 通道** 时，需进入完整审核流程，即：OPS审核 → 风控审核；
*   当所选通道 **仅为 AFP** 时，自动跳过风控审核，直接进入下一流程；
*   当商铺选择了多个通道时，只要其中 **包含任意非 AFP 通道**，都走完整审核流程。

因此，`待风控审核` 属于通用流程中的可选状态，并非所有商铺都会进入该状态。

#### 商铺审核主流程

| 流程顺序 | 流程             | 角色或触发方       | 对应状态         | 说明                                                              |
| :--- | :------------- | :----------- | :----------- | :-------------------------------------------------------------- |
| 1    | 创建             | OPS人员 / 外部平台 | 待审核          | 可以由 OPS 人员在后台创建商铺；也可以由平台侧客户自主创建或通过外部平台接口触发创建。创建来源通过上單來源字段记录。    |
| 2    | OPS审核中         | OPS人员        | 待审核          | OPS 人员在审核中，可以编辑资料，但不会变更状态。                                      |
| 3    | OPS审核结束，进入下一流程 | 系统 / 风控人员    | 待风控审核 / 审核通过 | 若商铺所选通道中包含非 AFP 通道，则流转到待风控审核；若商铺所选通道仅为 AFP，则自动跳过风控审核，直接流转到审核通过。 |
| 4    | 风控审核完成         | 风控人员         | 审核通过 / 审核不通过 | 仅在进入风控审核时才存在该步骤。风控审核通过后，商铺变为审核通过，并把启用状态变成启用；审核不通过则不会赋予启用状态。     |

#### 其他说明

1.  启用且审核通过的商铺，可通过【启用 / 停用】按钮改变启用状态，该按钮不会影响审核状态。
2.  后续点击【编辑】按钮时，走商铺资料修改待审核流程，不在上单审核流程中。
3.  创建人为创建该商铺数据的具体主体名称。若由 OPS 人员在后台创建，则显示对应 OPS 人员名称；若由外部平台触发创建，则显示该平台调用我方平台接口时传入的主体名称。
4.  商铺审核状态与企业审核状态为独立对象状态，互不替代；但在业务上可存在前置关系时，需由后端按规则控制是否允许继续创建或提交商铺。
5.  商铺审核通过后，表示商铺主体资料在本平台已通过当前阶段审核；是否进一步创建本地 MID、第三方渠道 MID 或触发 AFP 对接流程，由后续流程继续处理。

### 商铺列表页

#### 筛选区

##### 1. 功能定位

商铺管理列表页的筛选区，用于帮助运营人员在商铺数据量较多时，快速缩小查询范围，定位目标商铺，并进入后续查看、编辑、关联查询等操作。

##### 2. 交互形式

筛选区位于商铺管理列表页顶部，位于列表内容区上方，筛选区支持【展开 / 收起】交互：

*   默认为收起状态，仅展示两行的核心筛选项；
*   点击【展开】后，显示更多补充筛选项；
*   点击【收起】后，恢复为默认展示状态；
*   同类列表页的【展开 / 收起】按钮位置、顺序、样式需保持一致；
*   筛选区采用响应式栅格布局。桌面端每行固定展示 3 个筛选项，操作按钮区域固定在筛选区右下角。日期范围选择器按 1 个完整筛选项宽度处理。收起状态共展示 2 行，最多 6 个槽位，其中未用满的槽位保留对齐，不允许自动重排按钮位置。
*   收起状态下固定展示以下字段：商铺名称关键词、地区、商铺启用状态、商铺审核状态、服务通道、创建时间。其余字段仅在展开后显示。这几个字段也默认按顺序排在最前面。
*   其他后台列表页的筛选区，除筛选字段本身不同外，展开 / 收起规则、按钮顺序、布局方式、默认收起逻辑均与本页保持一致。

##### 3. 操作按钮

筛选区固定提供以下操作按钮，且按顺序排布：

*   **查询** （demo中不需要做此按钮交互）
*   **重置** （demo中不需要做此按钮交互）
*   **展开 / 收起**

##### 4. 筛选字段

| 字段      | 控件类型    | 说明                                     |
| :------ | :------ | :------------------------------------- |
| 商铺名称关键词 | 输入框     | 支持对商铺各名称（商铺名称，简称，英文名）识别信息进行模糊搜索        |
| 商铺编号    | 输入框     | 支持按商铺编号进行精确或模糊搜索                       |
| 地区      | 下拉单选    |                                        |
| 商铺启用状态  | 下拉单选    | 枚举值：启用、停用                              |
| 商铺审核状态  | 下拉单选    | 枚举值：待审核、审核通过、待风控审核、审核不通过               |
| 创建时间    | 日期范围选择器 |                                        |
| 上單來源    | 下拉多选    | 枚举值：CRM、DMO、SaaS、其他（预留）                |

| 營業鋪位    | 下拉单选    | 枚举值：地鋪、樓上、地庫、商場、活動、網店(只限Online）、其他（预留） |
| 联系电话    | 输入框     |                                        |
| 电邮地址    | 输入框     |                                        |

#### 列表区

列表区分为数据区和操作区两个部分组成，都置于同一行中，数据区从左到右排列数据。列表区按表格行展示。操作列固定位于最右侧，与数据列属于同一行。操作列内多个按钮按纵向排列，顶部对齐显示。若某行操作按钮较多导致该行高度增加，则整行所有单元格高度保持一致。

##### 列表显示字段

1.  商铺名称（本地语言）&#x20;
2.  商铺名称（英文）&#x20;
3.  商铺编号（SID）&#x20;
4.  所属企业名称&#x20;
5.  所属企业编号（CID）&#x20;
6.  地区&#x20;
7.  商铺类型&#x20;
8.  商铺启用状态&#x20;
9.  商铺审核状态&#x20;
10. 服务通道&#x20;
11. 门店联系电话&#x20;
12. 门店联系邮箱&#x20;
13. 营业国家&#x20;
14. 營業鋪位
15. 门店营业地址（原始全文）&#x20;
16. 特殊业务属性标签&#x20;
17. 上單來源&#x20;
18. 创建人&#x20;
19. 创建时间&#x20;
20. 更新时间&#x20;
21. 操作&#x20;

##### 布局要求

1.  各列表列按内容设置合理宽度，优先保证主要信息可读性。商铺名称列建议宽度 320-400px，所属企业名称列建议宽度 240-320px，其他文本列建议宽度 160-240px。允许表格出现横向滚动，但不允许因列宽过窄导致大部分内容无法识别。
2.  文本类单元格内容超长时允许换行展示，最多显示 3 行；超过 3 行的内容以省略号截断，并支持悬浮查看完整内容。

##### 操作区

操作区根据不同状态，显示不同的按钮，见下表：

| 启用状态 | 审核    | 列表页操作按钮     |
| :--- | :---- | :---------- |
| 空    | 待审核   | 查看详情        |
| 空    | 待风控审核 | 查看详情        |
| 启用   | 审核通过  | 查看详情、编辑、停用  |
| 停用   | 审核通过  | 查看详情、编辑、启用  |
| 空    | 审核不通过 | 查看详情、重新提交审核 |

##### 状态补充规则

*   只有审核通过的才会存在启用状态，待审核、待风控审核、审核不通过的数据不会有启用状态。
*   当商铺审核状态为“待审核”“待风控审核”或“审核不通过”时，商铺启用状态为空，不展示启用状态标签，也不允许执行启用 / 停用操作。
*   仅当商铺审核状态为“审核通过”时，列表页才允许展示“启用”或“停用”按钮。
*   点击【重新提交审核】后，该商铺审核状态变更为“待审核”，启用状态保持为空。
*   列表页仅提供轻量操作入口，审核通过 / 审核驳回 / 通知补件等审批类动作统一在详情页执行。

##### 操作按钮说明

| 按钮      | 功能说明                                                                          |
| :------ | :---------------------------------------------------------------------------- |
| 查看详情    | 点击进入详情页面                                                                      |
| 编辑      | 点击进入编辑页面                                                                      |
| 启用 / 停用 | 点击出现弹窗，提示文案【是否启用（或停用）该商铺】，提供操作：关闭 / 确认。点击【关闭】关闭弹窗，不改状态；点击【确认】把商铺的启用状态变成对应的状态。 |
| 重新提交审核  | 再次提交审核，把该商铺变成待审核状态。                                                           |

#### 功能区

提供两个操作按钮：

*   导出（demo中不需要做此按钮交互）
*   导入（demo中不需要做此按钮交互）

功能区位于列表区上方、筛选区下方，与列表标题同行，按钮靠右排列。后续其他列表页中的功能区，皆按此布局。

***

#### 与其他页面的关系要求

1.  商铺列表页中的字段名称、状态文案、标签样式，需要与商铺详情页、商铺编辑页保持一致。
2.  商铺列表页中的审核状态需完整体现底层流程口径，包括：待审核、待风控审核、审核通过、审核不通过。
3.  当商铺仅选择 AFP 通道时，OPS 审核通过后可自动跳过待风控审核状态；但列表页仍需兼容展示该状态，以支持非 AFP 或混合通道场景。
4.  商铺列表页中的设备数量、MID 数量为只读统计值，不在本页直接维护。
5.  本页为商铺对象的入口页，审批类完整动作在详情页执行，资料修改类动作通过编辑页进入资料修改待审核流程。

#### Demo实现要求

*   本页可基于本地 mock 数据实现；
*   筛选区需实现展开 / 收起的样式与布局；
*   demo中不需要实现查询、重置、导出、导入的真实功能；
*   列表区需根据不同状态展示不同操作按钮；
*   商铺审核状态与启用状态需按文档中的规则联动展示；
*   可通过 mock 数据模拟仅 AFP 通道、非 AFP 通道、混合通道三种商铺，以便验证待风控审核状态是否出现；
*   demo 中不要求真实后端接口，但字段结构与状态流转方式需符合后台系统设计思路。
