# AGENTS.md

## 1. 项目定位

这是一个用于生成 AFP CRM Demo 的前端原型仓库。

当前阶段目标不是开发真实生产系统，而是：

- 基于已确认需求文档生成可运行的前端 demo
- 用于内部评审、需求确认、交互验证
- 优先保证业务结构、字段口径、状态逻辑、页面关系正确
- 暂不要求真实后端接口

---

## 2. 当前任务范围

当前第一优先级任务只处理：

- 企业管理功能

包括以下页面：

1. 企业列表页
2. 企业详情页
3. 企业编辑页

本轮先不要主动扩展到：

- 商铺管理完整版
- MID 页面
- 设备独立管理页面
- 配置中心页面
- 其他未明确要求的新页面

如果页面中需要引用商铺、MID、设备，只允许作为企业页面中的关联摘要信息展示，不要擅自扩展成独立功能流。

## 4. 技术实现要求

默认技术方案：

- React
- TypeScript
- Ant Design
- 本地 mock data
- 前端路由
- 不需要真实后端接口

要求：

- 生成一个可运行的前端 demo
- 默认首页进入企业列表页
- 支持列表页、详情页、编辑页之间跳转
- 支持部分本地交互和状态模拟

除非有明确必要，不要退回原生 HTML 多页方案。

## 5. UI 风格总控要求

整体页面风格必须偏向：

- 海外成熟 CRM / SaaS 后台
- Professional
- Clean
- Structured
- Enterprise
- Efficient
- Minimal but not empty

必须遵守以下规则：

### 5.1 组件强约束

所有交互组件必须严格遵循 Ant Design 体系，包括但不限于：

- Button
- Input
- Select
- DatePicker
- Table
- Tag
- Badge
- Tabs（仅在明确允许时）
- Modal
- Drawer
- Pagination
- Tooltip
- Collapse
- Card
- Form
- Checkbox
- Radio
- Alert
- Steps

禁止自创与 Ant Design 明显不一致的组件风格。

### 5.2 颜色约束

如果仓库中提供了公司 Logo，则必须：

- 从 Logo 中提取主色和辅助色
- 主色用于主按钮、当前选中态、链接、重点标签、局部高亮
- 辅助色只能少量点缀
- 页面大面积背景仍然必须以白色、浅灰、深灰为主
- 不得脱离 Logo 色系另起一套颜色体系

禁止：

- 高饱和大面积铺色
- 渐变滥用
- 引入无关颜色
- 做成营销页或活动页

### 5.3 布局风格要求

布局必须接近国外成熟 CRM 后台，而不是传统国内密集型后台。

要求：

- 留白充足
- 模块边界清晰
- 字体层级分明
- 表格与筛选区结构规整
- 状态表达统一
- 只读信息强调可读性，不要伪装成可编辑表单

### 5.4 禁止出现的风格问题

必须避免：

- 中国式拥挤后台
- 低保真线框图感
- 大量彩色卡片
- 夸张阴影
- 过大圆角
- 装饰性图形泛滥
- 蓝紫渐变国际化模板感
- 自定义奇怪按钮和表单控件

## 6. 输出优先级

生成页面时，必须严格按以下优先级执行：

## 8. 审核与状态规则

系统底层流程需要兼容 AFP，必须严格遵守：

- 如果客户所选通道中包含非 AFP 通道，则需要进入风控审核
- 如果客户所选通道仅为 AFP，则自动跳过风控审核
- 如果选择多个通道，只要包含任意非 AFP 通道，仍然走完整审核流程

因此：

- `待风控审核` 是通用流程中的可选状态
- 不是所有企业都会进入 `待风控审核`

同时必须遵守：

- 只有审核通过的数据，才允许有启用状态
- 待审核、待风控审核、审核不通过时，启用状态为空
- 列表页中的启用 / 停用按钮，只能出现在审核通过状态下
- 编辑页提交走“资料修改待审核流程”，不是“创建审核流程”

## 9. 企业列表页硬性要求

企业列表页必须包含：

- 页面标题区
- 筛选区
- 功能区
- 列表区

### 9.1 筛选区规则

必须支持展开 / 收起，并严格遵守：

- 桌面端每行固定 3 个筛选项
- 收起状态展示两行
- 操作按钮固定在右下角
- 按钮顺序固定为：查询、重置、展开 / 收起
- 默认展示核心筛选项
- 其他字段展开后展示
- demo 中查询、重置、导出、导入不需要真实功能，但界面与交互入口要保留

### 9.2 列表区规则

列表字段、状态展示、操作按钮必须严格按需求文档实现，不允许自由增删。

列表页仅提供轻量操作入口：

- 查看详情
- 编辑
- 启用 / 停用
- 重新提交审核

审核通过 / 驳回 / 补件等审批类动作统一在详情页执行。

---

## 10. 企业详情页硬性要求

企业详情页必须：

- 使用单页长页面
- 所有区域都在同一页面连续展示
- 顶部使用 tags / 锚点跳转
- 不允许使用 tab 切换并卸载内容

固定区域必须包含：

1. 企业概览
2. 企业主键与关联
3. 企业名称信息
4. 企业主体与经营信息
5. 企业基础经营数据
6. 企业联系信息
7. 企业地址信息
8. 企业文件信息
9. 风控 / 业务信息
10. BL / AH 信息
11. AFP 对接信息
12. 拥有设备
13. 相关公司
14. 相关人员
15. 修改记录时间轴

### 10.1 AFP 对接信息区硬性要求

AFP 对接信息区必须采用：

- 概要信息 + 查看详情弹窗

不允许在页面正文中直接平铺全部 AFP 字段。

页面正文只显示每个 AFP 对象最关键的 2~3 个字段，其余字段全部收敛到弹窗中。

AFP 字段名必须使用 AFP 接口字段名，不允许改成中文。

## 11. 企业编辑页硬性要求

企业编辑页只允许编辑企业主体自身的非 AFP 数据。

### 11.1 可编辑范围

仅允许编辑：

- 企业主体基础资料
- 企业联系方式
- 企业经营属性
- 企业法务属性
- 本平台自维护字段
- 本平台备注说明类字段

### 11.2 不可编辑范围

不允许编辑：

- AFP 返回或同步字段
- AFP 状态字段
- AFP 主体识别字段
- AFP 相关开户信息
- 关联商铺数据
- 关联 MID 数据
- 系统生成字段
- 审核痕迹字段

### 11.3 页面组成

页面必须包含：

1. 基础信息区
2. 企业资料编辑区
3. AFP 相关信息展示区
4. 关联商铺信息展示区
5. 关联 MID 信息展示区
6. 底部操作区

### 11.4 提交流程

编辑提交时：

- 不直接覆盖正式数据
- 只模拟生成“企业资料修改待审核记录”
- 不直接修改审核状态与启用状态

## 12. 页面联动要求

必须实现以下基本联动：

- 列表页点击查看详情，进入详情页
- 列表页点击编辑，进入编辑页
- 详情页点击编辑，进入编辑页
- 编辑页提交后，模拟生成待审核修改记录
- 详情页审批 / 补件 / 启用 / 停用后，本地状态同步更新
- 列表页和详情页状态展示保持联动
- 时间轴同步增加记录

如果单次任务提示与 `AGENTS.md` 冲突，以 `AGENTS.md` 为准。
