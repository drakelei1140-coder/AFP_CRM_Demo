## 商户审核功能说明

### 商户审核-详情页

#### 1. 功能定位

商户审核详情页用于查看某一商户（MID）在审核流程中的完整资料，并承接商户审核流程中的主要审核动作。\
本页需要在同一个页面中集中展示：

*   商户自身资料
*   企业审核信息
*   商铺审核信息
*   渠道与 AFP Store 信息
*   结算与银行信息
*   产品与支付方式信息
*   特殊费率申请信息
*   修改记录时间轴

本页主要服务于以下目标：

1.  让 OPS、风控核查、风控初审、风控中审、总经理在一个页面内查看当前 MID 的完整审核资料；
2.  支持根据不同审核状态展示不同操作按钮；
3.  支持在审核过程中同时查看企业与商铺当前状态，并决定是否需要先完成企业或商铺审核；
4.  支持风控人员基于企业和商铺数据判断商户风险等级；
5.  支持承接商户审核流程中的核查、转交、退回、审核通过、拒绝签约、取消签约等动作。

***

#### 2. 页面整体交互形式

· 2.1 页面结构

商户审核详情页采用 **单页长页面** 形式，所有区域自上而下连续展示。\
不再使用传统 tab 容器切换内容。

· 2.2 顶部 tags 跳转区

页面顶部提供一排 tags 按钮，用于快速跳转到对应内容区域。

· 2.3 tags 交互规则

*   页面初始加载后默认定位在页面顶部；
*   点击某个 tag 后，页面滚动到对应区域标题位置；
*   当前滚动所在区域，对应 tag 可高亮；
*   所有区域都在同一页面中始终存在，不允许因为点击 tag 而切换或卸载内容；
*   后续其他详情页若采用同样交互方式，则整体规则保持一致。

· 2.4 建议的 tags 顺序

1.  商户概览
2.  主键与关联
3.  审核流转信息
4.  企业审核信息
5.  商铺审核信息
6.  渠道与 AFP Store 信息
7.  结算与银行信息
8.  产品与支付方式信息
9.  BA / Transfer Instrument / PM 信息
10. 特殊费率申请信息
11. 修改记录时间轴

***

#### 3. 页面顶部概览区

页面顶部先展示一块概览区，用于快速查看商户的关键信息和当前审核状态。

· 3.1 概览区显示内容

*   MID
*   CID
*   SID
*   渠道编码
*   服务编码
*   进件通道
*   商户审核状态
*   当前审核人
*   上單來源
*   创建人
*   创建时间
*   更新时间
*   AFP Store ID
*   AFP merchant account ID

· 3.2 展示要求

*   商户审核状态以状态标签形式展示；
*   当前审核人、MID、CID、SID、渠道编码、服务编码作为主要识别信息优先展示；
*   `AFP Store ID`、`AFP merchant account ID` 作为渠道执行识别信息展示；
*   若字段为空，统一显示 `-`。

#### 4. 页面顶部操作按钮

根据不同状态，在详情页提供对应的按钮。

| 商户审核状态  | 详情页操作按钮                                        |
| :------ | :--------------------------------------------- |
| 待审核     | 开始核查                                           |
| 基础资料审核  | 取消簽約、從已有商戶複製資料、轉交審核、遞交審核                       |
| 风控核查（1） | 开始核查                                           |
| 风控核查（2） | 通知BD（OS补件）、取消签约、风险评分风控核查、退回审核、转交审核、递交风控、基础资料评定 |
| 风控初级审核  | 通知BD（OS补件）、取消签约、退回审核、转交审核、审核通过、拒绝签约            |
| 风控中级审核  | 通知BD（OS补件）、取消签约、退回审核、转交审核、审核通过、拒绝签约            |
| 总经理审核   | 通知BD（OS补件）、取消签约、退回审核、转交审核、审核通过、拒绝签约            |
| 风控审核完成  | 无详情页操作按钮                                       |

#### 4.1 操作说明

【开始核查】——点击后进入当前节点的正式处理状态，并记录当前处理人。\
【取消簽約】——变更状态为取消签约，将在【取消签约商户】列表中查看。\
【從已有商戶複製資料】——原有功能，用于从已有商户复制相关资料。\
【轉交審核】——换人审核，出现弹窗，下拉选择其他人。\
【遞交審核】——基础资料审核完成后，提交到下一审核环节。\
【通知BD（OS补件）】——数据推到风控 OS 中，不用做 demo 交互。\
【风险评分风控核查】——填写或查看风险评分表。\
【退回审核】——风控初审、中审、总经理审核过程中点击，都会退回到【风控核查（1）】状态。\
【递交风控】——风控核查完成后，进入风控初级审核。\
【基础资料评定】——风控核查过程中完成基础资料评定。\
【审核通过】——根据风险等级决定流转到下一层审核，或流转到风控审核完成。\
【拒绝签约】——变更状态为拒绝签约，将在【拒绝签约商户】列表中查看。

#### 4.2 特别说明

*   `风控核查（1）` 和 `风控核查（2）` 在系统中实际定义为同一个状态 `风控核查`，这里只是因为操作按钮不同，所以在功能说明中拆开描述；
*   在没有点击【开始核查】前，系统未记录具体风控核查人员；
*   谁点击【开始核查】，谁负责后续核查；
*   其他核查人员不应看到不属于自己的 `风控核查（2）` 后续操作按钮。

#### 5. 主键与关联信息区

该区域展示 MID 表中 **主键 / 关联** 分组字段。

· 显示字段

*   MID
*   CID
*   SID
*   渠道编码
*   服务编码

· 展示要求

*   该区域以只读字段形式展示；
*   ID 类字段支持复制；
*   空值统一显示 `-`；
*   建议采用 2\~3 列的只读表单布局展示。

***

#### 6. 审核流转信息区

该区域用于展示当前商户在审核流程中的审核信息。

· 显示字段

*   商户审核状态
*   当前审核人
*   上單來源
*   创建人
*   创建时间
*   更新时间
*   当前风险等级（如有）
*   当前风险评分表状态（如有）
*   是否存在在途特殊费率申请
*   特殊费率申请结果
*   特殊费率申请完成时间

· 展示要求

*   该区域建议采用只读表单 + 状态标签形式展示；
*   审核状态字段需要突出显示；
*   当前审核人为空时统一显示 `-`；
*   特殊费率申请相关字段建议作为独立小块展示。

#### 6.1 流转特别说明

*   当所选通道中包含 **非 AFP 通道** 时，需进入完整审核流程；
*   当所选通道 **仅为 AFP** 时，自动跳过 `销售主管审核`、`风控核查`、`风控初级审核`、`风控中级审核`、`总经理审核`，直接进入 `风控审核完成`；
*   但即使是仅 AFP 通道，也仍然要先到 `风控审核完成`，然后再根据特殊费率申请逻辑自动流转到 `已签约`；
*   `销售主管审核` 状态虽然保留，但系统强逻辑自动跳过；
*   `风控审核完成` 状态的存在，是为了兼容原系统中的特殊费率申请并行流程。

#### 7. 企业审核信息区

该区域用于展示当前 MID 关联企业（CID）的资料摘要、审核状态以及可执行操作。\
该区域的作用不是替代企业详情页，而是让当前处理商户审核的人员，可以在商户详情页内同步判断企业是否已经完成当前应做的审核。

#### 7.1 业务原则

*   企业和商铺在商户详情页中 **分开审核**；
*   不允许把企业和商铺合并成一个统一审核动作；
*   对于 **非 AFP 通道** 的商户，如果企业或商铺 **任意一个** 未完成风控审核，则 **不允许完成商户审核流程**；
*   风控人员判断商户风险等级时，需要参考企业与商铺数据，因此必须在本页可见企业摘要信息与状态；
*   OPS 审核商户时，如果企业当前也轮到 OPS 审核，则同样要在本页可见并可处理。

#### 7.2 显示内容

该区域建议展示企业详情页中的关键摘要字段，而不是平铺完整企业详情页全部内容。

建议展示以下内容：

##### 7.2.1 基础识别与状态摘要

*   企业主显示名称
*   CID
*   企业 LE ID
*   企业启用状态
*   企业审核状态
*   進件通道(必須)
*   上單來源
*   創建時間
*   更新時間

##### 7.2.2 企业审核关键字段摘要

*   公司模式
*   公司結構
*   行業分類 / 業務性質 / 產品服務
*   MCC Code
*   進階MCC Code
*   小微商戶
*   SME 設定
*   法律地位
*   僱員人數
*   注冊資本
*   企业税务申报分类
*   企业税务业务类型
*   平均每月營業額
*   每宗交易平均金額
*   每宗交易最大交易額
*   預計每年交易宗數
*   過往拒付比例
*   過往退款比例
*   提前收款日數
*   貨物 / 服務最短送達時間
*   貨物 / 服務最長送達時間
*   是否對公
*   合約期
*   是否NGO
*   商戶營業狀態（企業口徑）

##### 7.2.3 企业风险与业务信息摘要

*   風控類型
*   風險等級
*   審核類型
*   服務類型
*   合作夥伴推薦人
*   合作夥伴編號
*   交單備註
*   是否高額商戶

#### 7.3 按钮与交互

当企业当前状态轮到当前登录角色处理时，提供以下按钮：

*   【查看详情】——在新的浏览器标签页打开企业详情页
*   【审核】——在当前商户详情页内打开企业审核抽屉，直接对企业执行审核

#### 7.4 审核按钮显示规则

##### 7.4.1 对 OPS 人员

如果企业当前状态轮到 OPS 审核，则显示：

*   【查看详情】
*   【审核】

##### 7.4.2 对风控人员

如果企业当前状态轮到风控审核，则显示：

*   【查看详情】
*   【审核】

##### 7.4.3 其他情况

如果当前企业状态不轮到当前登录角色处理，则：

*   可展示【查看详情】
*   不显示【审核】按钮

#### 7.5 审核抽屉要求

点击【审核】后，在当前页右侧打开 **企业审核抽屉**。

抽屉要求：

*   抽屉标题明确标识为【企业审核】
*   抽屉中展示企业审核所需的关键字段摘要
*   抽屉底部展示当前阶段对应的审核操作按钮
*   审核完成后，刷新商户详情页中的企业状态展示
*   抽屉仅处理企业审核，不处理商铺审核和商户审核

#### 7.6 阻塞提示规则

当企业未完成当前应完成的审核时，在该区域顶部提示：

*   若当前轮到 OPS：`企业尚未完成 OPS 审核，可能影响后续商户审核流转`
*   若当前轮到风控：`企业尚未完成风控审核，商户审核暂不可完成`

#### 9. 商户审核阻塞规则

该区域用于说明企业、商铺状态对商户审核的影响。

##### 9.1 规则说明

对于 **非 AFP 通道** 的商户：

*   如果企业和商铺 **任意一个** 未完成风控审核，则商户 **不允许完成审核流程**
*   只有当企业和商铺 **都完成风控审核** 后，商户才允许继续走完商户审核流程

##### 9.2 页面提示方式

当存在阻塞时，在商户详情页顶部操作区下方展示阻塞提示条，例如：

*   `当前企业未完成风控审核，暂不可完成商户审核`
*   `当前商铺未完成风控审核，暂不可完成商户审核`
*   `当前企业与商铺均未完成风控审核，暂不可完成商户审核`

##### 9.3 对按钮的影响

在存在阻塞时：

*   商户详情页中涉及“完成当前商户审核阶段”的按钮应禁用或拦截提交
*   企业和商铺区域中的【审核】按钮仍可正常使用
*   风控人员应优先完成企业或商铺审核，再返回继续处理商户审核

##### 9.4 AFP 特别说明

对于 **仅 AFP 通道** 的商户：

*   企业、商铺、MID 都会自动跳过风控审核流程
*   此时不需要应用上述“企业 / 商铺风控未完成阻塞商户风控完成”的规则
*   但页面仍需兼容展示企业与商铺状态，用于辅助判断整体数据情况

#### 17. 与其他页面的关系要求

1.  商户审核详情页中的字段名称、状态文案、标签样式，需要与商户审核列表页、商户审核编辑页保持一致。
2.  本页为商户审核流程的主要处理页面，完整审核动作都在本页执行。
3.  本页中的 MID 相关字段需与 Excel 中已定义的 MID 字段口径保持一致，不允许擅自替换成未定义字段。
4.  企业审核信息区、商铺审核信息区，分别引用企业详情页和商铺详情页中的关键字段摘要，但不替代完整详情页。
5.  在商户详情页中，企业和商铺必须 **分开审核**，不得合并处理。
6.  对于非 AFP 通道的商户，若企业或商铺任意一个未完成风控审核，则商户不得完成审核流程。
7.  仅 AFP 通道的 MID，仍需支持展示 `风控审核完成` 这一系统态，以兼容自动流转到 `已签约` 的逻辑。

*   demo 中不需要实现“一键导出”“通知BD（OS补件）”等原有功能的真实交互；
*   demo 中需体现不同状态下按钮差异，以及企业 / 商铺状态对商户审核的阻塞关系。
