## 企业管理功能说明

### 企业详情页

#### 1. 功能定位

企业详情页用于查看某一企业的完整资料。\
本页需要在同一个页面中集中展示企业主体信息、经营信息、地址信息、联系信息、文件信息、AFP 对接信息、设备信息、关联公司、关联人员以及修改记录时间轴。

本页主要服务于以下目标：

1.  让运营人员在一个页面内完整查看企业全部资料；
2.  不再通过 tab 切换内容，减少来回切换成本；
3.  支持快速跳转到指定内容区域；
4.  支持人工核对企业字段与 AFP 对接字段之间的映射结果；
5.  支持查看企业当前状态、下属商铺数量、MID 数量、相关设备和关系信息。

#### 2. 页面整体交互形式

· 2.1 页面结构

企业详情页采用 **单页长页面** 形式，所有区域自上而下连续展示。\
不再使用传统 tab 容器切换内容。

· 2.2 顶部 tags 跳转区

页面顶部提供一排 tags 按钮，用于快速跳转到对应内容区域。

· 2.3 tags 交互规则

*   页面初始加载后默认定位在页面顶部；
*   点击某个 tag 后，页面滚动到对应区域标题位置；
*   当前滚动所在区域，对应 tag 可高亮；
*   所有区域都在同一页面中始终存在，不允许因为点击 tag 而切换或卸载内容；
*   后续其他详情页若采用同样交互方式，则整体规则保持一致。

· 2.4 建议的 tags 顺序

1.  企业概览
2.  企业主键与关联
3.  企业名称信息
4.  企业主体与经营信息
5.  企业基础经营数据
6.  企业联系信息
7.  企业地址信息
8.  企业文件信息
9.  风控/业务信息
10. BL / AH 信息
11. AFP 对接信息
12. 拥有设备
13. 相关公司
14. 相关人员
15. 修改记录时间轴

#### 3. 页面顶部概览区

页面顶部先展示一块概览区，用于快速查看企业的关键信息和当前状态。

· 3.1 概览区显示内容

*   企业主显示名称
*   CID
*   企业 LE ID
*   企業啟用狀態
*   企業審核狀態
*   下属商铺数量
*   MID 数量
*   進件通道(必須)
*   上單來源
*   創建時間
*   更新時間

· 3.2 展示要求

*   企业启用状态、企业审核状态以状态标签形式展示；
*   下属商铺数量、MID 数量以统计卡片或醒目数字形式展示；
*   企业名称、CID、LE ID 作为主要识别信息优先展示；
*   若字段为空，统一显示 `-`。

· 3.3 操作按钮

根据不同的状态，在详情页也提供对应的按钮

| 启用状态 | 审核    | 详情页操作按钮                     |
| :--- | :---- | :-------------------------- |
| 空    | 待审核   | 编辑，审核通过（OPS），审核驳回，通知补件（OPS） |
| 空    | 待风控审核 | 编辑，审核通过（风控），审核驳回，通知补件（风控）   |
| 启用   | 审核通过  | 编辑、停用                       |
| 停用   | 审核通过  | 编辑、启用                       |
| 空    | 审核不通过 | 重新提交审核                      |

【审核通过（OPS）】——

*   若该企业所选通道中包含非 AFP 通道，则流转到待风控审核状态；
*   若该企业所选通道仅为 AFP，则自动跳过风控审核，直接流转到审核通过状态，并首次自动启用。

按钮作用：

【编辑】——点击进入编辑页面 【审核通过】——弹出弹窗，填写审核意见，再点击确认把状态变成审核通过&#x20;

【审核通过（OPS）】——流转到待风控审核状态

【审核通过（风控）】——流转到审核通过状态

【审核驳回】——弹出弹窗，填写审核意见，再点击确认把状态变成审核不通过&#x20;

【通知补件(OPS)】——点击出现补件弹窗，需要运营人员自定义需要补件的内容，可以添加多个内容。提供按钮，关闭/确认，点击确认后，状态不变，修改记录时间轴中会显示谁在什么时候操作了需要补件，补件内容包括什么（上面运营人员自定义的内容） 。数据流转到OPS-补件

【通知补件(风控)】——点击出现补件弹窗，需要运营人员自定义需要补件的内容，可以添加多个内容。提供按钮，关闭/确认，点击确认后，状态不变，修改记录时间轴中会显示谁在什么时候操作了需要补件，补件内容包括什么（上面运营人员自定义的内容）。数据流转到风控-补件

【启用】or【停用】——点击出现弹窗，提示文案【是否启用（或停用）该企业】，提供操作：关闭/确认。点击【关闭】关闭弹窗，不改状态；点击【确认】把企业的启用状态变成对应的状态。&#x20;

【重新提交审核】——再次提交审核，把该企业变成待审核状态。

#### 13. AFP 对接信息区

该区域用于展示 **企业字段映射到 AFP 后的对接信息**。\
为避免详情页内容过长，本区域不在页面正文中直接展示全部 AFP 映射字段，而是采用 **“概要信息 + 查看详情弹窗”** 的交互方式。

&#x20;13.1 展示原则

*   页面正文中，每个 AFP 对象区域仅展示少量关键字段；
*   每个对象区域都提供 **【查看详情】** 按钮；
*   点击【查看详情】后，弹出对应对象的详情弹窗；
*   弹窗中再展示该对象的完整 AFP 映射字段；
*   AFP 区域内的字段名统一使用 **AFP 接口字段名（英文）**；

*   页面正文不直接平铺完整映射表

#### 18. 页面布局要求

· 18.1 整体布局

*   页面采用单列纵向布局；
*   各区域之间保留明显间距；
*   每个区域有独立标题；
*   顶部 tags 区建议吸顶；
*   页面不再以 tab 容器切换内容。

· 18.2 区域展示形式

不同区域可以采用不同展示形式：

*   基础字段：只读表单布局
*   附件类字段：文件列表
*   AFP 对接信息：映射表格
*   拥有设备 / 相关公司 / 相关人员：表格
*   修改记录：纵向时间轴组件

· 18.3 字段显示规则

*   空值统一显示 `-`
*   文本过长允许换行
*   附件类字段显示文件名或查看按钮
*   ID 字段支持复制
*   状态字段统一使用标签样式

#### 19. 与其他页面的关系要求

1.  企业详情页中的字段名称、状态文案、标签样式，需要与企业列表页、企业编辑页保持一致。
2.  页面顶部必须突出显示企业启用状态、企业审核状态、下属商铺数量、MID 数量。
3.  AFP 对接信息区中的字段名必须使用 AFP 接口字段名，不允许改为中文。
4.  本页为只读详情页，不在本页直接编辑企业字段；若后续需要编辑，应通过编辑页或资料修改待审核流程处理。
5.  相关公司、相关人员、拥有设备、修改记录时间轴都属于本页固定区域，不通过 tab 切换隐藏。
