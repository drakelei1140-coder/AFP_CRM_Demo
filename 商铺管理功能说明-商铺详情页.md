## 商铺管理功能说明

### 商铺详情页

#### 1. 功能定位

商铺详情页用于查看某一商铺的完整资料。\
本页需要在同一个页面中集中展示商铺主体信息、经营信息、地址信息、联系信息、文件信息、AFP 对接信息、终端设备信息、关联企业、关联 MID 以及修改记录时间轴。

本页主要服务于以下目标：

1.  让运营人员在一个页面内完整查看商铺全部资料；
2.  不再通过 tab 切换内容，减少来回切换成本；
3.  支持快速跳转到指定内容区域；
4.  支持人工核对商铺字段与 AFP 对接字段之间的映射结果；
5.  支持查看商铺当前状态、所属企业、终端设备数量、MID 数量等关键信息。

#### 2. 页面整体交互形式

· 2.1 页面结构

商铺详情页采用 **单页长页面** 形式，所有区域自上而下连续展示。\
不再使用传统 tab 容器切换内容。

· 2.2 顶部 tags 跳转区

页面顶部提供一排 tags 按钮，用于快速跳转到对应内容区域。

· 2.3 tags 交互规则

*   页面初始加载后默认定位在页面顶部；
*   点击某个 tag 后，页面滚动到对应区域标题位置；
*   当前滚动所在区域，对应 tag 可高亮；
*   所有区域都在同一页面中始终存在，不允许因为点击 tag 而切换或卸载内容；
*   后续其他详情页若采用同样交互方式，则整体规则保持一致。

· 2.4 建议的 tags 顺序

1.  商铺概览
2.  商铺主键与关联
3.  商铺名称信息
4.  商铺主体与经营信息
5.  商铺基础经营数据
6.  商铺联系信息
7.  商铺地址信息
8.  商铺文件信息
9.  风控 / 业务信息
10. Store / Terminal 信息
11. AFP 对接信息
12. 终端设备
13. 关联企业
14. 关联 MID
15. 修改记录时间轴

#### 3. 页面顶部概览区

页面顶部先展示一块概览区，用于快速查看商铺的关键信息和当前状态。

· 3.1 概览区显示内容

*   SID
*   所属企业名称

*   商铺启用状态

*   终端设备数量
*   MID 数量
*   進件通道(必須)
*   上單來源
*   創建時間
*   更新時間

· 3.2 展示要求

*   商铺启用状态、商铺审核状态以状态标签形式展示；
*   终端设备数量、MID 数量以统计卡片或醒目数字形式展示；
*   商铺名称、SID、所属企业、Store ID 作为主要识别信息优先展示；
*   若字段为空，统一显示 `-`。

· 3.3 操作按钮

根据不同的状态，在详情页也提供对应的按钮。

| 启用状态 | 审核    | 详情页操作按钮                     |
| :--- | :---- | :-------------------------- |
| 空    | 待审核   | 编辑，审核通过（OPS），审核驳回，通知补件（OPS） |
| 空    | 待风控审核 | 编辑，审核通过（风控），审核驳回，通知补件（风控）   |
| 启用   | 审核通过  | 编辑、停用                       |
| 停用   | 审核通过  | 编辑、启用                       |
| 空    | 审核不通过 | 重新提交审核                      |

按钮作用：

【编辑】——点击进入编辑页面

【审核通过（OPS）】——

*   若该商铺所选通道中包含非 AFP 通道，则流转到待风控审核状态；
*   若该商铺所选通道仅为 AFP，则自动跳过风控审核，直接流转到审核通过状态，并首次自动启用。

【审核通过（风控）】——流转到审核通过状态，并首次自动启用。

【审核驳回】——弹出弹窗，填写审核意见，再点击确认把状态变成审核不通过。

【通知补件（OPS）】——点击出现补件弹窗，需要运营人员自定义需要补件的内容，可以添加多个内容。提供按钮：关闭 / 确认。点击确认后，状态不变，修改记录时间轴中会显示谁在什么时候操作了需要补件，补件内容包括什么。数据流转到 OPS-补件。

【通知补件（风控）】——点击出现补件弹窗，需要运营人员自定义需要补件的内容，可以添加多个内容。提供按钮：关闭 / 确认。点击确认后，状态不变，修改记录时间轴中会显示谁在什么时候操作了需要补件，补件内容包括什么。数据流转到风控-补件。

【启用】or【停用】——点击出现弹窗，提示文案【是否启用（或停用）该商铺】，提供操作：关闭 / 确认。点击【关闭】关闭弹窗，不改状态；点击【确认】把商铺的启用状态变成对应的状态。

【重新提交审核】——再次提交审核，把该商铺变成待审核状态。

#### 13. AFP 对接信息区

该区域用于展示 **商铺字段映射到 AFP 后的对接信息**。\
为避免详情页内容过长，本区域不在页面正文中直接展示全部 AFP 映射字段，而是采用 **“概要信息 + 查看详情弹窗”** 的交互方式。

13.1 展示原则

*   页面正文中，每个 AFP 对象区域仅展示少量关键字段；
*   每个对象区域都提供 **【查看详情】** 按钮；
*   点击【查看详情】后，弹出对应对象的详情弹窗；
*   弹窗中再展示该对象的完整 AFP 映射字段；
*   AFP 区域内的字段名统一使用 **AFP 接口字段名（英文）**；
*   字段值取当前平台映射字段对应的值；
*   可增加“来源字段”列，用于辅助运营核对；
*   本区域只读，不允许在详情页直接编辑。

13.2 Store 映射信息

该区域用于展示商铺对应的 Store 映射结果。

—— 页面正文显示内容

页面正文中仅展示以下字段：

*   `id`
*   `reference`
*   `shopperStatement` 或 `description`

—— 页面正文操作

*   提供 **【查看详情】** 按钮

—— 查看详情弹窗

点击【查看详情】后，弹出 **Store 映射详情弹窗**，弹窗中展示完整 Store 映射字段。

—— Store 弹窗建议展示字段

*   `id`
*   `reference`
*   `description`
*   `shopperStatement`
*   `status`
*   `salesChannels[]`
*   `address.line1 / street`
*   `address.line2 / street2`
*   `address.city`
*   `address.stateOrProvince`
*   `address.postalCode`
*   `address.country`
*   `splitConfiguration`
*   `externalReference`
*   `metadata`

13.4 页面展示形式

AFP 对接信息区建议按 **2 个独立子区域** 展示：

1.  Store 映射信息
2.  Terminal / Device 映射信息

每个子区域展示形式统一如下：

*   区域标题
*   关键字段只读展示
*   右上角提供 **【查看详情】** 按钮

—— 弹窗展示形式

点击【查看详情】后，弹窗内建议使用表格展示完整字段，推荐结构如下：

| AFP字段名 | 当前值 | 来源字段 |
| ------ | --- | ---- |

—— 弹窗交互要求

*   弹窗标题需明确区分对象类型，例如：
    *   Store 映射详情
    *   Terminal / Device 映射详情
*   弹窗支持滚动查看全部字段；
*   弹窗宽度建议较大，便于展示长字段名和长内容；
*   弹窗底部提供关闭按钮；
*   弹窗仅用于查看，不提供编辑能力。

13.5 页面展示收敛规则

为避免详情页信息过载，AFP 对接信息区统一采用以下收敛规则：

*   页面正文只展示每个 AFP 对象最关键的 2\~3 个字段；
*   其余字段全部收敛到详情弹窗中；
*   页面正文不直接平铺完整映射表；
*   弹窗中展示完整 AFP 字段与映射值，用于需要时人工核对。

#### 14. 终端设备区

该区域用于展示该商铺当前拥有的设备信息。\
显示口径按设备型号与数量汇总，不展示价格。

· 显示内容

*   设备型号
*   设备分类
*   数量
*   绑定状态

· 展示要求

*   同型号设备合并汇总展示；
*   不显示价格、费用、收据等信息；
*   建议使用表格形式展示；
*   数量为只读统计值。

#### 15. 关联企业区

该区域用于展示与当前商铺直接关联的企业信息。

· 显示内容

*   企业名称
*   企业编号（CID）
*   企业启用状态
*   企业审核状态
*   主联系人
*   关系说明

· 展示要求

*   建议使用只读信息卡片或表格形式展示；
*   当前一期默认展示主要所属企业；
*   若暂无关联企业，显示空状态；
*   企业编号支持复制，可提供【查看详情】入口跳转到企业详情页。

#### 16. 关联 MID 区

*   服务通道
*   当前状态

#### 17. 修改记录时间轴区

该区域用于展示该商铺的重要修改记录。

· 展示内容

每一条时间轴记录建议包含：

*   操作时间
*   操作人
*   操作对象
*   操作类型
*   变更摘要
*   备注

· 展示要求

*   按时间倒序展示；
*   最新记录在最上方；
*   可区分商铺资料修改、状态流转、AFP 对接更新、设备变更、MID 关联变更等类型；
*   变更摘要建议显示修改前后差异的简版说明；
*   若无记录，显示空状态。

#### 18. 页面布局要求

· 18.1 整体布局

*   页面采用单列纵向布局；
*   各区域之间保留明显间距；
*   每个区域有独立标题；
*   顶部 tags 区建议吸顶；
*   页面不再以 tab 容器切换内容。

· 18.2 区域展示形式

不同区域可以采用不同展示形式：

*   基础字段：只读表单布局
*   附件类字段：文件列表
*   AFP 对接信息：映射表格
*   终端设备 / 关联企业 / 关联 MID：表格
*   修改记录：纵向时间轴组件

· 18.3 字段显示规则

*   空值统一显示 `-`
*   文本过长允许换行
*   附件类字段显示文件名或查看按钮
*   ID 字段支持复制
*   状态字段统一使用标签样式

***

#### 19. 与其他页面的关系要求

1.  商铺详情页中的字段名称、状态文案、标签样式，需要与商铺列表页、商铺编辑页保持一致。
2.  页面顶部必须突出显示商铺启用状态、商铺审核状态、所属企业、终端设备数量、MID 数量。
3.  AFP 对接信息区中的字段名必须使用 AFP 接口字段名，不允许改为中文。
4.  本页为只读详情页，不在本页直接编辑商铺字段；若后续需要编辑，应通过编辑页或资料修改待审核流程处理。
5.  终端设备、关联企业、关联 MID、修改记录时间轴都属于本页固定区域，不通过 tab 切换隐藏。
6.  当商铺仅选择 AFP 通道时，OPS 审核通过后可直接跳过待风控审核状态；但详情页仍需兼容展示“待风控审核”对应的按钮和状态，以支持非 AFP 或混合通道场景。
