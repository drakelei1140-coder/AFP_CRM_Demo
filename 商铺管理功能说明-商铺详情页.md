## 商铺管理功能说明

### 商铺详情页

#### 1. 功能定位

商铺详情页用于查看某一商铺的完整资料。\
本页需要在同一个页面中集中展示商铺主体信息、经营信息、地址信息、联系信息、文件信息、终端设备信息、关联企业、关联 MID 以及修改记录时间轴。

本页主要服务于以下目标：

1.  让运营人员在一个页面内完整查看商铺全部资料；
2.  不再通过 tab 切换内容，减少来回切换成本；
3.  支持快速跳转到指定内容区域；
4.  支持查看商铺当前状态、所属企业、终端设备数量、MID 数量等关键信息；
5.  当前详情页不再承载 AFP 映射信息展示。

***

#### 2. 页面整体交互形式

##### 2.1 页面结构

商铺详情页采用 **单页长页面** 形式，所有区域自上而下连续展示。\
不再使用传统 tab 容器切换内容。

##### 2.2 顶部 tags 跳转区

页面顶部提供一排 tags 按钮，用于快速跳转到对应内容区域。

##### 2.3 tags 交互规则

*   页面初始加载后默认定位在页面顶部；
*   点击某个 tag 后，页面滚动到对应区域标题位置；
*   当前滚动所在区域，对应 tag 可高亮；
*   所有区域都在同一页面中始终存在，不允许因为点击 tag 而切换或卸载内容；
*   顶部导航区需置顶悬浮，页面滚动时不移出可视区域；
*   tags 的视觉风格禁止使用深色底色标签样式，统一使用浅底色 + 深色文字风格；
*   后续其他详情页若采用同样交互方式，则整体规则保持一致。

##### 2.4 建议的 tags 顺序

1.  商铺概览
2.  商铺主键与关联
3.  商铺名称信息
4.  商铺主体与经营信息
5.  商铺基础经营数据
6.  商铺联系信息
7.  商铺地址信息
8.  商铺文件信息
9.  风控 / 业务信息
10. 终端设备
11. 关联企业
12. 关联 MID
13. 修改记录时间轴

#### 3. 页面顶部概览区

页面顶部先展示一块概览区，用于快速查看商铺的关键信息和当前状态。

##### 3.1 概览区显示内容

*   商铺主显示名称
*   SID
*   所属企业名称
*   所属企业编号（CID）
*   商铺启用状态
*   商铺审核状态
*   终端设备数量
*   MID 数量
*   進件通道(必須)
*   上單來源
*   創建時間
*   更新時間

##### 3.2 展示要求

*   商铺启用状态、商铺审核状态以状态标签形式展示；
*   终端设备数量、MID 数量以统计卡片或醒目数字形式展示；
*   商铺名称、SID、所属企业作为主要识别信息优先展示；
*   不再显示 Store ID 等 AFP / Store 返回识别信息；
*   若字段为空，统一显示 `-`。

##### 3.3 操作按钮

根据不同状态，在详情页也提供对应按钮。

| 启用状态 | 审核    | 详情页操作按钮                     |
| :--- | :---- | :-------------------------- |
| 空    | 待审核   | 编辑、审核通过（OPS）、审核驳回、通知补件（OPS） |
| 空    | 待风控审核 | 编辑、审核通过（风控）、审核驳回、通知补件（风控）   |
| 启用   | 审核通过  | 编辑、停用                       |
| 停用   | 审核通过  | 编辑、启用                       |
| 空    | 审核不通过 | 重新提交审核                      |

按钮作用：

【编辑】——点击进入编辑页面

【审核通过（OPS）】——

*   若该商铺所选通道中包含非 AFP 通道，则流转到待风控审核状态；
*   若该商铺所选通道仅为 AFP，则自动跳过风控审核，直接流转到审核通过状态，并首次自动启用。

【审核通过（风控）】——流转到审核通过状态，并首次自动启用。

【审核驳回】——弹出弹窗，填写审核意见，再点击确认把状态变成审核不通过。

【通知补件（OPS）】——点击出现补件弹窗，需要运营人员自定义需要补件的内容，可以添加多个内容。提供按钮：关闭 / 确认。点击确认后，状态不变，修改记录时间轴中会显示谁在什么时候操作了需要补件，补件内容包括什么。数据流转到 OPS-补件。

【通知补件（风控）】——点击出现补件弹窗，需要运营人员自定义需要补件的内容，可以添加多个内容。提供按钮：关闭 / 确认。点击确认后，状态不变，修改记录时间轴中会显示谁在什么时候操作了需要补件，补件内容包括什么。数据流转到风控-补件。

【启用】or【停用】——点击出现弹窗，提示文案【是否启用（或停用）该商铺】，提供操作：关闭 / 确认。点击【关闭】关闭弹窗，不改状态；点击【确认】把商铺的启用状态变成对应的状态。

【重新提交审核】——再次提交审核，把该商铺变成待审核状态。

#### 16. 页面收敛规则

本页需要明确去掉以下旧内容，不再显示：

*   【AFP 对接信息】区域
*   【Store / Terminal 信息】区域中的 AFP / Store 识别与映射内容
*   Store ID、Store Reference、Store Status 等 AFP / Store 返回识别信息

但这并不意味着删除原字段定义，只是当前商铺详情页中不显示 AFP 映射内容；对应字段后续可在商户进件功能的详情页中展示。

#### 17. 字段对齐要求

除去本次明确移除的 AFP / Store 展示内容外，其余商铺详情页字段需要重新对齐 Excel 中商铺对象的全字段口径。\
不得因为页面收敛而漏掉本应在商铺详情页展示的商铺主体字段。

#### 18. Demo实现要求

*   本页可基于本地 mock 数据实现；
*   顶部导航区需置顶悬浮；
*   tags 禁止使用深色底色风格；
*   页面保持单页长页面结构；
*   不再出现 AFP 对接信息区；
*   不再出现 Store / Terminal 信息区；
*   设备统计统一放在【终端设备】区域；
*   demo 中不要求真实后端接口，但字段结构与状态流转方式需符合后台系统设计思路。
