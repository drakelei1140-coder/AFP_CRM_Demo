## 商铺管理功能说明

### 新增商铺页面

#### 页面定位

新增商铺页面用于创建新的商铺主体数据。\
该页面服务于 OPS 人员或平台侧来源创建商铺的场景，用于录入商铺相关字段，并提交进入商铺创建审核主流程。

新增商铺页面需满足以下原则：

*   新增时可填写全部商铺相关字段；
*   新增页面需要选择已经存在的企业作为关联；
*   不能在新增商铺页面直接新增企业；
*   页面字段口径需与商铺详情页、商铺列表页保持一致；
*   页面用于创建新数据，不等同于编辑页；
*   新增提交后，进入商铺创建审核流程，而不是资料修改待审核流程；
*   页面顶部需要有和详情页一样的悬浮导航菜单；
*   页面底部提供固定悬浮操作区。

#### 页面组成

新增商铺页面由以下几个区域组成：

1.  页面顶部概览区
2.  顶部悬浮导航菜单
3.  所属企业选择区
4.  商铺资料填写区
5.  AFP 相关信息展示区
6.  底部悬浮操作区

#### 页面顶部概览区

建议展示内容：

*   页面标题：新增商铺
*   页面说明：用于创建新的商铺主体资料，保存后进入待审核状态
*   当前操作人
*   上單來源（如有默认值可回显）

#### 顶部悬浮导航菜单

页面顶部需要提供与详情页一致风格的悬浮导航菜单，用于快速跳转到对应内容区域。

建议导航项包括：

1.  商铺概览
2.  商铺主键与关联
3.  商铺名称信息
4.  商铺主体与经营信息
5.  商铺基础经营数据
6.  商铺联系信息
7.  商铺地址信息
8.  商铺文件信息
9.  风控 / 业务信息
10. Store / Terminal 信息
11. AFP 对接信息

交互要求：

*   页面滚动时，导航菜单保持悬浮；
*   点击导航项后，页面滚动到对应区域标题位置；
*   当前滚动所在区域，对应导航项高亮；
*   所有区域都在同一页面内，不通过 tab 切换隐藏内容。

#### 所属企业选择区

新增商铺时，必须先选择已经存在的企业作为关联对象。

展示内容建议包括：

*   当前已选企业名称
*   当前已选企业编号（CID）
*   当前已选企业状态摘要
*   按钮：【选择企业】
*   按钮：【从企业复制信息】

交互规则：

*   新增商铺页面不能直接新增企业；
*   未选择企业前，页面可继续填写草稿内容，但不能执行保存；
*   已选择企业后，页面才建立企业与商铺的关联关系。

##### 选择企业

点击【选择企业】后，打开企业选择弹窗。

弹窗要求：

*   弹窗中展示当前所有企业正式数据列表，非企业草稿；
*   列表字段可与企业列表页字段一致；
*   提供筛选区，仅支持：
    *   按企业编号搜索
    *   按企业名称模糊搜索
*   操作区仅提供【选择】按钮；
*   可以选择的企业不分状态。

点击【选择】后：

*   将对应企业设为当前新增商铺的关联企业；
*   回显企业名称、企业编号等摘要信息；
*   不自动创建企业数据；
*   不允许在本页修改企业正式资料。

##### 从企业复制信息

当已选择企业后，提供按钮【从企业复制信息】。

功能规则如下：

*   复制固定映射字段；
*   只覆盖这些被定义为“可复制字段”的内容；
*   不影响商铺独有字段；
*   若当前商铺页已填写内容，点击后按“全部覆盖这些映射字段”处理。

可复制字段示例：

*   企业名称 → 商铺名称
*   企业地址 → 门店地址 / 营业地址
*   企业联系电话 → 商铺联系电话
*   企业邮箱 → 商铺联系邮箱

#### AFP 相关信息展示区

该区域用于展示 AFP 侧创建后才会产生的字段占位信息。

展示原则：

*   新增阶段不允许手工录入 AFP 返回字段；
*   该区域仅展示“待生成”“待同步”或空值状态；
*   不提供输入框、下拉框、开关等可编辑控件；
*   不允许通过新增页直接伪造 Store / Terminal 等 AFP 返回结果。

可展示的占位信息示例：

*   Store ID
*   Store Status
*   AFP 同步状态
*   终端设备同步状态

#### 必填项校验规则

*   必填字段未填写时，对应字段项后方显示红字提醒；
*   点击【存成草稿】时，不校验必填项；
*   点击【存成草稿】时，也不校验是否已经选择企业；
*   点击【保存】前，需要判断商铺必填字段；
*   未通过必填校验时，【保存】按钮置灰且不能点击；
*   未选择企业时，【保存】按钮置灰且不能点击。

#### 草稿箱规则

##### 草稿箱基础规则

*   商铺资料草稿箱与其他草稿箱相互独立；
*   商铺新增草稿只保存商铺表单内容，不带上已选企业；
*   每个操作员只能查看自己创建的草稿数据，不能查看别人的草稿数据。

##### 从草稿复制

点击【从草稿复制】按钮后，打开【商铺资料草稿箱】弹窗。

弹窗要求：

*   以表格形式展示当前操作人曾经记录过的商铺信息草稿；
*   表格字段可与商铺列表页字段一致；
*   操作区提供两个按钮：【选择】/【删除】。

操作说明：

*   点击【删除】可删除对应草稿；
*   点击【选择】可把选定草稿中的商铺信息复制到当前新增商铺页面；
*   复制为全部覆盖当前页面中的商铺表单内容；
*   复制完成后，草稿箱中的原草稿不会消失。

##### 存成草稿

点击【存成草稿】时：

*   将当前已经录入的商铺表单内容保存到【商铺资料草稿箱】；
*   不校验必填项；
*   不校验是否已选择企业；
*   不生成正式商铺数据；
*   不进入审核流程。

#### 底部悬浮操作区

页面底部增加一个悬浮且一直置于底部的操作区。

提供以下按钮：

*   从草稿复制
*   存成草稿
*   保存
*   取消

##### 按钮说明

| 按钮    | 功能说明                                      |
| :---- | :---------------------------------------- |
| 从草稿复制 | 打开商铺资料草稿箱弹窗，选择草稿后全部覆盖当前页面商铺表单内容           |
| 存成草稿  | 将当前已录入的商铺表单内容保存到商铺资料草稿箱，不校验必填项，也不校验是否已选企业 |
| 保存    | 校验必填字段及企业选择情况，校验通过后创建正式商铺数据               |
| 取消    | 放弃本次新增，返回上一页或商铺列表页                        |

#### 保存成功后的结果

*   点击【保存】后，系统进行必填项校验；
*   同时校验是否已经选择企业；
*   校验通过后，创建一笔正式商铺数据；
*   保存成功后，该商铺数据变成 **待审核** 状态；
*   保存成功后，跳转到商铺详情页；
*   后续是否进入待风控审核，仍按既有服务通道规则判断。

#### 与编辑页的区别

新增商铺页面与商铺编辑页需明确区分：

*   新增页用于创建新商铺；
*   编辑页用于修改已有商铺；
*   新增页提交后进入创建审核流程；
*   编辑页提交后进入资料修改待审核流程；
*   新增页不展示既有正式数据与历史修改痕迹；
*   编辑页需要展示已有只读字段、关联对象、设备摘要与 AFP 已返回数据。

#### Demo实现要求

*   本页可基于本地 mock 数据实现；
*   可编辑字段支持基础表单交互；
*   顶部悬浮导航菜单需实现跳转功能；
*   底部悬浮操作区需固定在页面底部；
*   企业选择弹窗需体现按编号和名称筛选能力；
*   【从企业复制信息】需体现固定映射字段覆盖逻辑；
*   草稿箱弹窗需体现【选择】/【删除】操作；
*   点击【保存】时，demo 中可模拟创建一笔待审核商铺数据，并跳转到商铺详情页；
*   demo 中不要求真实后端接口，但字段结构与状态流转方式需符合后台系统设计思路。
