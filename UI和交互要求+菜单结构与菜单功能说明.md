## UI和交互要求

*   UI规范请读取codex\_crm\_style\_prompt.md
*   整个后台分成几个区域：
    | 区域         | 说明                                                                                                                   |
    | :--------- | :------------------------------------------------------------------------------------------------------------------- |
    | 顶部形象区      | 顶部大概高度为80px，最左侧显示Kpay logo，右侧显示选择地区（下拉）、选择语言（下拉）、登录头像+登录名、设置入口按钮、消息通知按钮。一直置于浏览器顶部，不会因为鼠标滚动而移出浏览器，自适应于浏览器。            |
    | 左侧菜单区      | 所有页面通用同一个菜单，菜单展开的结构不会因为跳转页面而改变菜单展开的结构。整体菜单区域置于浏览器最左侧，且上下自适应于浏览器，不会因为鼠标滚动而移动。菜单中的内容超过浏览起的高度时，可以单独在菜单区域中上下滚动而查看到所有的内容。 |
    | 右侧内容区——列表页 | 包括筛选区、功能区、列表区。要求两个区域的布局分开处理，筛选区见下面筛选区的交互说明，列表区见列表区的交互说明。部分功能的列表页没有功能区，如果有，功能区都放在筛选区和列表区之间，提供操作按钮。                    |
    | 右侧内容区——详情页 | 见详情页的交互说明。                                                                                                           |
*

### 功能说明

*   对应菜单有下级菜单时，点击该菜单不指向具体功能页面，而是展开/收起对应的下级菜单，例如：【商户管理】【资料修改待审核】
*   对应菜单没有下级菜单时，点击进入对应的功能页面，详情见下表

| 菜单名称         | 点击跳转页面           |
| :----------- | :--------------- |
| 企业管理         | 企业列表页            |

| 企业资料修改待审核    | 企业资料修改待审核单列表页    |
| 商铺资料修改待审核    | 商铺资料修改待审核单列表页    |
| 商户资料修改待审核    | 商户资料修改待审核单列表页    |
